How to exchange data between components

Data Flow - Parent to Children

Sending data from parent to children is very similar with React js. You bind props to the component tag and receive the data on ngOnInit by asigning that to a new variable with the input decorator. @Input.

Exemple @Input() usage - Step by step

  1. Create a new variable on topic1 component with a string value
  2. Create a new component and use it´s selector here on topic1 html file.This new component will be the child.
  3. Set a property bing on child selector with the value as the variable name you want to pass to child component.
  4. Open the gate to receive the variable with @Input() on child component. Declare the property binding as a variable on child component.That´s ít.

Data Flow - Child to parent

To pass data from child to parent in react you must sento to child component a function and set args so parent component can access them . That´s how you can send data from child to parent in react.

But, what about Angular? How does it work?Well.. lets see.

There is two ways to pass data from child to parent component on Angular:



1- @ViewChild Decorator

Exemple @ViewChild() usage - Step by step

First thing you should realize is that on the HTML file of the child component you don´t have a selector of the parent component. so you can´t bind any property on selector. Because there is no parent selector on child HTML to bind. Like react, passing data from child to parent is mabe by using methods.But how it work?

  1. Create a variable inside the child component with the data you wnat to pass to parent component.
  2. In order to use @ViewChild() decorator it must be imported from @angular/core module as you imported the @input decorator before.
  3. The @ViewChild() decorator will be imported on the parent component.(The parent will see the child.)
  4. You have to pass, as an arg to @ViewChild() decorator , the child component you want to see and get data from.
  5. In order to pass the child component as an argument to @ViewChild decorator, you must import the component to parent component .ts file.
  6. Remember, react do not have dependeny injection, but Angular does. Once you inject the component on the buil in decorator , ViewChild(), all the propertys and methods of the child became accessible from the parent component.
  7. To look on console if you are receiving the data from child yoou must use ngAfterOninit life cicle hook. That´s because if you use the constructor or ngOnInit , the child component does not exists yet and will be display undefined on that life cicle momento.
  8. Just access child properties by using this. variable name and the child property or method.

Displaing data from child component:

{{messageFromChild}}



2- @Output Decorator & Event Emitter

Exemple and usage - Step by step

Ok. In React we pass data from child to parent by using args at a function passed as a prop. In Angular, the first way uses a similar approach. The parent component receives the child component as a arg on the @ViewChild decorator and set it to a variable so parent class can have access to all data and methods. The second way is when an event accours. Let´s see how it works.

  1. Create a variable on child component so you can share it to parent component.
  2. Import output decorator and eventemitter on child component from @angular/core module.
  3. Create Output decorator and asign a new instance of EventEmitter to the variable.
  4. inside the click method on children component you have to emmit the event with the data you want to send.
